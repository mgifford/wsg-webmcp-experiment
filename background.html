Experiment | Background | W3C Web Sustainability Guidelines

Why this experiment exists

The W3C Web Sustainability Guidelines provide practical guidance for designing, building, operating, and governing more sustainable digital products and services. Today, that guidance is primarily consumed as a human-readable document.

This experiment asks a different question: what happens when a web standard can also expose its guidance as structured tools that AI agents can discover and call?

WebMCP allows a web page to register tools with names, descriptions, input schemas, and executable functions. In this project, the WSG data is loaded from the published guidelines JSON file and exposed through experimental tools such as search, tag lookup, and statistics.

How WebMCP could help people build more sustainable products

A product team often needs sustainability guidance at the moment decisions are being made: during discovery, design, procurement, development, content planning, infrastructure review, or reporting. The challenge is that standards documents can be long, dense, and difficult to apply under time pressure.

WebMCP could help by allowing an AI assistant, browser tool, development environment, or audit workflow to ask the WSG site for structured guidance directly from the source. Instead of relying on stale training data or scraping a document, an agent could call a tool such as wsg.search or wsg.list_by_tag and receive current guidance from the published WSG data.

For example, a team could ask for guidance related to performance, accessibility, procurement, hosting, privacy, or content. An AI assistant could then return relevant WSG guidelines, success criteria, benefits, and supporting resources. This would not replace professional judgement, but it could make the guidance easier to find and apply.

The useful pattern is not “AI replaces standards.” The useful pattern is: standards expose structured, authoritative guidance that AI systems can use more accurately.

Example use cases

  • Design review: identify WSG criteria related to reducing unnecessary user journeys, distraction, deceptive patterns, or inaccessible design choices.
  • Development support: surface guidance related to JavaScript, CSS, assets, performance, compatibility, and low-bandwidth access.
  • Procurement: generate draft sustainability requirements for vendors, with links back to the relevant WSG guidance.
  • Auditing: create draft audit questions based on selected guidelines and success criteria.
  • Reporting: help teams prepare draft conformance or implementation notes, while making clear that human review is required.
  • Education: help practitioners discover related guidance, benefits, tags, and resources without needing to know the structure of the WSG document in advance.

What WebMCP adds beyond a normal web page

The normal web page works in any modern browser. Firefox can load the JSON, run the search interface, and display results because those features are plain JavaScript.

WebMCP adds a separate layer. In browsers that support WebMCP, the page can expose selected functions as structured tools for AI agents. Those tools can be inspected, tested, and invoked by compatible browser features, extensions, or agent systems.

In other words, Firefox can use the human interface. Chrome with WebMCP enabled can also expose the machine interface.

Current limitations

  • This is an unofficial experiment.
  • It does not certify WSG conformance.
  • It does not replace human review.
  • WebMCP is still experimental and browser support is limited.
  • The browser cannot directly tell a web page which internal Chrome flags are enabled.
